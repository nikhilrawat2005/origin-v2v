## ✨ Features

| Module | Description |
|---|---|
| 🏠 Landing Page | Hero, categories, benefits, testimonials |
| 🔐 Authentication | Email/password + Google sign-in, forgot password |
| 👤 Profile | Education, skills, interests, income — powers eligibility matching |
| 📊 Dashboard | Stats, saved opportunities, upcoming deadlines |
| 🔍 Explore | Search + filters over the live opportunity database |
| 📄 Opportunity Detail | Bookmark, share, set reminder, AI eligibility explanation |
| 🔖 Saved Opportunities | Bookmarked listings in one place |
| 🗂️ Application Tracker | Kanban-style board per application type (Job, Internship, Scholarship, etc.), each with its own stages and terminal states |
| 💰 Wallet | Upload and organize documents (resume, certificates, awards, projects, portfolio, ID) via Cloudinary |
| 🗓️ Calendar | Deadlines and tracked-application dates in one view |
| 👥 Community | Peer discussion space |
| 🏢 Organization Dashboard | View for partner organizations |
| 🔔 Notifications | In-app alerts, generated by a rule-based automation engine (zero AI cost) |
| 🛠️ Admin Panel | Restricted to a hard-coded allow-list, enforced in the client, the route, and Firestore rules |
| 🤖 AI Hub | Resume analysis, eligibility explanations, and other AI-assisted career tools |
| 📥 Automated Ingestion | Daily cron job that pulls new opportunities from trusted feeds (Grants.gov, Devpost, ioscholarships) and scraped sources (AnitaB, SWE, AAUW, Outreachy), normalizes and dedupes them, and auto-approves or queues them for review |
| 📧 Deadline Emails | Standalone backend service that emails users when a tracked deadline is 7, 3, or 1 day(s) away |

## 🛠️ Tech Stack

**Frontend**
- **Framework**: Next.js 16 (App Router) + TypeScript
- **Styling**: TailwindCSS v4
- **Animations**: Framer Motion
- **Forms**: React Hook Form + Zod
- **Icons**: Lucide React

**Backend / Data**
- **Auth + DB**: Firebase Authentication + Firestore (rules-enforced)
- **File storage**: Cloudinary (wallet documents)
- **AI providers**: Groq and OpenRouter, routed through a shared AI service client
- **Scheduled ingestion**: Vercel Cron (`/api/ingest`, daily) → `src/lib/ingestion/`
- **Deadline email service**: standalone Node/Express service (`/backend`) using `node-cron` + Nodemailer, scheduled independently of Vercel

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/nikhilrawat2005/origin-v2v.git
cd origin-v2v
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Create `.env.local` in the project root. See [Environment Variables](#-environment-variables) below for the full list.

### 4. Run the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### 5. (Optional) Run the deadline email backend

```bash
cd backend
npm install
cp .env.example .env
npm run dev
```

See [backend/README.md](backend/README.md) for details on Gmail App Passwords and the Firebase service account.

## 🔐 Admin Access

Admin role is enforced in three layers (defense-in-depth):

1. `src/context/AuthContext.tsx` — auto-assigns/self-heals `role: "admin"` on login for allow-listed emails only.
2. `src/app/dashboard/admin/page.tsx` — redirects anyone whose email isn't on the allow-list, regardless of their stored role.
3. `firestore.rules` — the `role` field on a `users/{uid}` document can only be set to `"admin"` by an allow-listed email. This is the layer that actually matters; the other two are UX.

Edit the list in `src/lib/adminConfig.ts` and redeploy `firestore.rules` to change admin access.

## 📥 Opportunity Ingestion Pipeline

`src/lib/ingestion/` powers the automated feed that keeps the opportunity database fresh:

- **Trusted feeds** (Grants.gov, Devpost, ioscholarships) — official RSS/APIs, high confidence, auto-approved.
- **Scraped sources** (AnitaB, SWE, AAUW, Outreachy) — general web pages, parsed with AI, always landed as `pending` for human review.
- Runs daily via Vercel Cron hitting `GET /api/ingest`, authorized by `CRON_SECRET` (sent automatically by Vercel, or passed manually as `?secret=` for testing).
- Listings are normalized and deduped before being written to Firestore.
